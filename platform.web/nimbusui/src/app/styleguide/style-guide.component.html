Document structure is the underlying foundation that assist screen readers in describing what is displayed to the user. These are some best practices for coding element attributes that help define elements for assisted technology. The goal is to provide an organized presentation of elements that enable easy navigation throughout the application and describe the current element in focus and what actions are available. Leveraging HTML native behaviors of elements helps simplify these directions.

    ARIA Landmarks

  • Role

    role is a property that describes a purpose of a section or element. Two important roles for sections are shown below. Navigation and Main roles help define the purpose of elements inside each section and helps the user navigate to the disired information. 

    <div id="nav" role="navigation">
    navigation goes here
</div>

<div id="main" role="main">
    main content displayed here
</div>

    Aria-label

    aria-label is a description that is relayed to the screen reader user. This can be very helpful when multiple navigation areas within a single page.

    <div id="nav" role="navigation" aria-label="main navigation">
  <ul>
    <li>Home</li>
    <li>Products</li>
    <li>Support</li>
  </ul>
</div>
        
<div id="nav" role="navigation" aria-label="case navigation">
  <ul>
    <li>Case Overview</li>
    <li>Technical Specifications</li>
    <li>Customer Reviews</li>
  </ul>
</div>

    In this case the screen reader user will hear something like navigation landmark, main navigation, then navigation landmark, product navigation

  • H1-H5 heading structure

    Use headings to define the page structure

    Headings are used to break up the content into a hierarchical structure much like an outline. Where sighted users can skim a page quickly by reading the visual headlines, a screen reader user can also skim a page by quickly traversing the headings. To semantically define headings simply use heading tags to label the beginning of each section.

    This both creates a semantic structure for your page and it allows screen reader users to be able to navigate by heading and jump from section to section.

    <h1>Our News Site</h1>
  <h2>World News</h2>
  <h2>National News</h2>
    <h3>Hot Topics</h3>
    <h3>Politics</h3>
  <h2>Science</h2>
    <h3>Health</h3>
    <h3>Environment</h3>
    <h3>Technology</h3>
  <h2>Entertainment</h2>

  • Tables

    Data tables must also have a caption or a summary.

    Caption

    The caption element identifies the table. It can be thought of as a title or a heading for a table.

    <table>
  <caption>Schedule for the week of March 6</caption>
  <tr>
    <th>
</table>

    Summary

    The summary can perform two different functions. It is allowable to have the summary perform both functions at the same time.

    • it can describe the layout of the table to help screen reader users know how to navigate the table
    • it can describe in narrative form what information the table is trying to convey
    <table summary="This is a description of what this table is displaying.">
  <tr>
</table>
  • Forms - Textbox, Select & Textarea --Using for and id attributes

    This method explicitly ties a label to an input element by matching the for attribute in the label tag with the id attribute in the input element tag.

    //input type=text

<label for=last-name>Last Name</label> 
<input id=last-name type=text name=last-name />


    //select

<label for=color>Choose the color you want</label>
<select id=color name=select>
  <option value=1>Green</option>
  <option value=2>Blue</option>
  <option value=3>Red</option>
</select>


    //textarea

<label for=feedback>Please leave your feedback:</label><br />
<textarea id=feedback name=feedback></textarea>
  • Forms - SINGLE CHECKBOX & Radio button

    Nesting an input element inside a label tag inherently makes label-input relationship

    This method ties a label to an input element by wrapping the input element with the label tag. The "for" attribute is not necessary. 

    <label>
  <input type=checkbox name=last-name />
  Last Name
</label>
  • MULTIPLE RELATED CHECKBOXES

    a collection of related check boxes needs to included in a fieldset and have a legend. This helps visually identify related elements plus provides a description for how these items are related. 

    <fieldset>
 <legend>Select your days of availability:</legend>
 <label>
   <input type=checkbox name=availability value=monday />
   Monday
 </label>
 <label>
   <input type=checkbox name=availability value=tuesday />
   Tuesday
 </label>
 <label>
   <input type=checkbox name=availability value=wednesday />
   Wednesday
 </label>
</fieldset>
  • Buttons

    Buttons are inherently accessible as long as the you provide an appropriate value attribute. 

    <input type=submit 
       name=submit 
       value=Submit this form
/>

    Image Buttons need an appropriate ALT attribute. 

    <input type=image 
       name=submitbutton 
       src=accessibilitybutton.jpg 
       alt=Submit this form 
/>
